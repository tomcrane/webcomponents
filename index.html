Canvas Panel Developer Narrative - Intro

Intro | Version 1 | Version 2 | Viewer 1

This is a series of experiments to get a feel for what Canvas Panel actually is.

I think that there are many components in evidence at https://canvas-panel.digirati.com/styleguide/, above and below the level of Canvas Panel. I think that the component I'm looking for is somewhere around the level of CanvasRepresentation in the current implementation. That doesn't mean that I wouldn't use all those other components in putting an application together - as we have done with V&A etc.

But if I want a super-powerful canvas renderer outside of any particular set of tools or framework, that will do the really hard scene evaluation and rendering, what are its interfaces? How do I program against it?

The questions I'm trying to answer are:

  • What did I want CP to do for me?
  • What did I want to manage myself?
  • How does it collaborate with other libraries?

In other words, can we be sure that our code is in the right places?

This uses a couple of implementations of something called Canvas Panel that don't really do much. You'll have to imagine that this component is offering a nice deep zoom interface, is capable of rendering AV, is capable of being given annotations to render, fires events as the user interacts, etc. This is an exercise in determining the boundaries and how the component feels right when used in many different frameworks. I've made CP a web component, as a simple way of encapsulating it. Obviously this demo is not a good cross browser solution, but again I'm more concerned with the feel of coding against it, whatever it actually is. Making it a web component made that easier to do.
